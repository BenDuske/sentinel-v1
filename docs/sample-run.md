# Sample run — proof it works (offline)

This is a **real, captured** end-to-end run of Sentinel v1 with **no LLM reachable** (the
configured endpoint pointed at a dead port, so `llm: offline`). It proves the differentiator —
the grounded rule layer + deterministic fallbacks — produces a defensible, demoable result with
zero external dependencies.

Reproduce:

```bash
# point at a dead endpoint to force offline mode
SENTINEL_ASSUME_CONSENT=1 SENTINEL_LLM_BASE=http://127.0.0.1:59999/v1 \
  uvicorn sentinel.app:app --app-dir src --port 8231
```

```

                  UPS units at risk."
```

## With an LLM present

When a local OpenAI-compatible endpoint **is** reachable, the same flow uses it and
**reconciles** it with the rule layer (higher wins). Verified live against a local llama.cpp
server: for "Warehouse fire, building ablaze, two injured" the LLM judged `high`, the rule layer
floored at `critical` (fire), and the final reconciled severity was **critical**.
